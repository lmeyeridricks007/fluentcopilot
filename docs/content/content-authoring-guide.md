# Content authoring guide (Stage 5)

## Where files live

| Path | Purpose |
|------|---------|
| `content/courses/<id>/course.manifest.json` | Lists module JSON paths; loader builds a full `Course`. |
| `content/modules/<moduleId>/module.json` | Canonical `moduleSchema`: catalogs + embedded `lessons[]`. |
| `content/review-items/<moduleId>.json` | `{ "reviewItems": [...] }` — generated by `extract-review-items`. |
| `content/lessons/` | Optional standalone lesson JSON (import into module via tooling). |
| `content/grammar/`, `content/vocab/` | Optional shared spine fragments (future splits). |

## Lesson JSON (final shape)

Lessons **must** parse with `lessonSchema` (`src/lib/schemas/lesson.schema.ts`):

- **id**, **moduleId**, **title**, **lessonType**, **order**, **cefrLevel**, **durationEstimate**
- **grammarTargets**: string ids → must exist on parent module `grammarTargets[]`
- **vocabTargets**: string ids → must exist on parent module `vocabTargets[]`
- **canDoStatements**: ≥ 1 short EN outcome
- **steps**: ≥ 1 discriminated step (`lessonStep.schema.ts`)
- **metadata**: object (use `contentFormatVersion: 1` at module/course level)
- **reviewItemRefs**: optional; pipeline can fill via `tools/extract-review-items.ts --patch-module`
- **mistakeFocus**: optional taxonomy tags for review prioritisation

### Steps

Each step includes **`type`** plus optional **`prompt`**, **`content`**, **`interactionConfig`**, **`feedbackConfig`**, **`exercises`**, **`correctAnswers`**, **`hints`**, **`audioRefs`**, **`metadata`**.

Interactive steps should carry **`feedbackConfig`** (hint / incorrectFeedback / errorTags) where the engine records mistakes.

## Naming

- Kebab-case ids; lesson ids like `a2-m01-l02-morning-evening`.
- Review item ids: `rev-{lessonId}-vt-{vocabId}`, `rev-{lessonId}-gt-{grammarId}`, `rev-{lessonId}-ph-{slug}` (see `reviewItemExtractor.ts`).

## Workflow

1. Edit `module.json` (or generate via LLM using `tools/generate-lesson.ts` / `generate-module.ts` prompts).
2. `npx tsx tools/validate-content.ts --module ... [--review ...]`
3. `npx tsx tools/extract-review-items.ts <module.json> --out content/review-items/<id>.json --patch-module`
4. `npx tsx tools/validate-content.ts --manifest content/courses/nl-a2/course.manifest.json --review content/review-items/...`

## Lesson engine bundle

The schema player expects `{ moduleCatalog: { grammarTargets, vocabTargets }, lesson }`. Use `toSchemaLessonBundle(module, lessonId)` from `schemaLessonBundle.ts` (client-safe; no Node `fs`).
